📌 问题：出售优化是什么，我只有求购

【简答】
出售优化是用于自动管理您已上架出售商品的功能。
如果您只需要求购功能，出售优化已经被禁用了。

════════════════════════════════════════════════════════════════════════════

【什么是出售优化?】

出售优化（Sales Optimization）是对已经上架出售的商品进行的自动化管理：

1️⃣ 市场价监控
   └─ 实时检查市场上该商品的最低售价
   └─ 如果您的出售价高于市场价，自动降价

2️⃣ 利润保护
   └─ 降价前验证利润率是否满足8%的最小要求
   └─ 防止亏损出售

3️⃣ 库存清理
   └─ 如果市场价太低，无法保证利润，自动下架

【出售优化工作流程示例】

获取在售商品
  ↓
对每件商品:
  ├─ 获取市场最低售价
  ├─ 对比当前出售价
  │
  ├─ 如果高于市场价:
  │  ├─ 计算利润率
  │  ├─ 利润≥8% → 自动降价 ✓
  │  └─ 利润<8% → 自动下架 ✗
  │
  └─ 如果≈市场价:
     └─ 保持不变

【例子】
出售价: ¥980
成本价: ¥850
市场价: ¥950
利润: (950-850)/850 = 11.76% > 8%
操作: 自动降价到 ¥950 ✓

【启用/禁用出售优化的方法】

假设您未来需要启用出售优化:

步骤1: 编辑 cmd/price-monitor/main.go
       找到第 206-217 行

步骤2: 原来的代码:
       // 【已禁用】出售商品优化功能
       // 如需启用，请取消注释下面的代码
       /*
       for _, saleItem := range saleItems {
       	err := pm.processSaleItem(ctx, saleItem)
       	...
       }
       */

步骤3: 改为:
       // 4. 处理每件在售商品
       for _, saleItem := range saleItems {
       	err := pm.processSaleItem(ctx, saleItem)
       	...
       }

步骤4: 重新编译并启动:
       cd cmd/price-monitor
       go build -o ../../bin/price-monitor main.go impl.go
       pkill price-monitor
       ./bin/price-monitor -token "xxx" -interval 3600
